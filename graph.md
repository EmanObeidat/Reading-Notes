# Graphs:
Graphs are data structures used in computer science and mathematics to depict relationships between objects. They are made up of vertices  and edges that connect pairs of vertices.
Graphs are commonly utilized in domains such as computer science, data analysis, network analysis, and optimization. They give a versatile means of modeling and analyzing complicated systems and relationships.
```
here are some key terms and concepts related to graphs in computer science and mathematics:

Graph: A data structure consisting of a set of nodes (vertices) connected by edges. Graphs are used to represent relationships between entities.

Node (Vertex): A fundamental unit in a graph, representing an entity or a point. Nodes are connected by edges.

Edge: A connection between two nodes in a graph that represents a relationship or link between them.

Directed Graph (Digraph): A graph in which edges have a direction, meaning they go from one node to another. This direction can be one-way or bidirectional.

Undirected Graph: A graph in which edges have no direction, meaning they simply represent a connection between nodes without a specific source or destination.

Weighted Graph: A graph in which each edge has an associated weight or cost, representing a numerical value that could represent distances, costs, or any other relevant metric.

Degree of a Node: The number of edges connected to a particular node. In a directed graph, there's an in-degree (incoming edges) and an out-degree (outgoing edges).

Path: A sequence of nodes connected by edges, where each node is adjacent to the next node in the sequence.

Cycle: A path in which the first and last nodes are the same, forming a closed loop.

Connected Graph: A graph in which there is a path between every pair of nodes. In a directed graph, there might be different levels of connectivity, such as weakly connected or strongly connected.

Tree: A connected, acyclic (no cycles) undirected graph, often used for hierarchical structures or decision-making processes.

DAG (Directed Acyclic Graph): A directed graph that does not contain any cycles. DAGs are commonly used to represent processes, dependencies, and flowcharts.

Adjacency Matrix: A matrix representation of a graph where rows and columns correspond to nodes, and the matrix entries indicate whether there is an edge between the nodes.

Adjacency List: A list that stores the neighbors of each node in a graph. It's a more memory-efficient way to represent sparse graphs.

Graph Traversal: The process of visiting all the nodes in a graph in a systematic manner. Common traversal algorithms include Breadth-First Search (BFS) and Depth-First Search (DFS).


```
## Types of Graphs:
```
Graphs can be categorized into various types based on their characteristics and properties. Here are some common types of graphs:

Undirected Graph: A graph in which edges have no direction. The relationship between nodes is symmetric.

Directed Graph (Digraph): A graph in which edges have a specific direction from one node to another. The relationship between nodes can be asymmetric.

Weighted Graph: A graph in which edges have associated weights or costs, representing numerical values that indicate some kind of measurement, distance, or importance.

Unweighted Graph: A graph in which edges do not have any associated weights or costs. It only represents connections between nodes.

Connected Graph: A graph in which there is a path between any pair of nodes. It is possible to move from any node to any other node using the edges.

Disconnected Graph: A graph that is not connected, meaning there are nodes or groups of nodes that are not reachable from each other.

Cyclic Graph: A graph that contains at least one cycle, which is a closed path that begins and ends at the same node.
```
